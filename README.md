# Trading Application

Многомодульное Java-приложение для автоматической и тестовой торговли криптовалютами на бирже ByBit с использованием
аналитики паттернов.

## 📋 О проекте

**Trading Application** — настольное приложение для автоматической торговли Bitcoin на бирже ByBit.  
Система анализирует рыночные данные, выявляет торговые паттерны и на их основе совершает сделки.  
Ключевая особенность — наличие **тестового режима**, позволяющего проверять эффективность паттернов без риска потери
реальных средств.

### Возможности

- Реальная торговля на ByBit через официальный API
- Тестовая торговля на исторических и реальных данных (без отправки ордеров на биржу)
- Анализ рыночных данных и поиск торговых паттернов
- Два режима работы: консольный и с графическим интерфейсом (через REST API)
- Единая база данных для хранения всех событий
- Поддержка нескольких бирж и брокеров (ByBit, Tbank и др.)

## 🏗 Архитектура

Приложение построено как **модульный монолит**:

- Один процесс, одна JVM
- Чёткое разделение на функциональные модули внутри одного приложения
- В будущем возможно выделение модулей в микросервисы без переписывания логики

### Common

Содержит общие компоненты, используемые всеми модулями:

- Интерфейсы
- DTO
- Утилиты, константы, настройки подключения к БД
- Общие исключения

### Exchange-Connector

Модуль для работы с API бирж и брокеров. Каждая реализация вынесена в отдельный подмодуль для изоляции зависимостей и
SDK.

**Структура:**

- `exchange-connector-core` — абстрактные классы, общая логика обработки ошибок, ретраи
- `exchange-connector-bybit` — реализация для ByBit (официальный Java SDK)
- `exchange-connector-tbank` — реализация для Т-банка (брокерский API)

### Market-Data

Модуль для получения рыночных данных (котировки, свечи, стаканы). Также разделён на подмодули по источникам данных.

**Структура:**

- `market-data-core` — общая логика, кэширование, обработка ошибок
- `market-data-bybit` — получение котировок с ByBit
- `market-data-tbank` — получение котировок из Т-банка

### Core (ядро приложения)

#### Trader-Common

Общая логика для реальной и тестовой торговли:

- Валидация ордеров
- Расчёт комиссий и размеров позиций
- Мапперы для преобразования данных
- Базовые абстрактные классы для торговых исполнителей

#### Trader (реальная торговля)

Модуль реальной торговли. Получает котировки от `market-data`, сравнивает с паттернами из БД и при их обнаружении
формирует и отправляет ордера через `exchange-connector`.

**Структура пакетов:**

- `bybit` — реализация для ByBit (использует `exchange-connector-bybit`)
- `tbank` — реализация для Т-банка (использует `exchange-connector-tbank`)

Записывает информацию о сделках в таблицу `trades`.

#### io.github.ijlijapol.Test-Trader (тестовая торговля)

Модуль тестовой торговли. Получает те же рыночные данные, но **не отправляет реальные ордера**. Вместо этого симулирует
сделки и записывает их в таблицу `test_trades`. Позволяет оценить эффективность паттернов без финансового риска.

**Структура пакетов:**

- `bybit` — симуляция торговли на ByBit
- `tbank` — симуляция торговли в Т-банке

### Analyzer

Аналитический модуль. В реальном времени анализирует поток котировок от `market-data`, выявляет заданные торговые
паттерны и сохраняет информацию о них в таблицу `patterns`.

**Технологии:** многопоточность (`@Async`), паттерны анализа данных (технические индикаторы, свечные паттерны).

**Структура пакетов:**

- `common` — общие алгоритмы и индикаторы
- `bybit` — специфичная для ByBit логика
- `tbank` — специфичная для Т-банка логика

### Interaction (модули взаимодействия с пользователем)

#### Ui-Api (фасад)

Предоставляет единый набор сервисов для всех типов интерфейсов:

Использует модули `core` и `analyzer`. Все UI-модули зависят только от `ui-api`, что изолирует их от деталей реализации
ядра.

#### Console-Ui

Консольный интерфейс для управления приложением. Реализует интерфейс из `common` и вызывает методы `ui-api`.

**Возможности:**

- Просмотр текущих котировок
- Включение/отключение торговых модулей
- Просмотр статистики по сделкам и паттернам
- Управление настройками анализатора

**Технологии:** Spring Shell или простой `Scanner`.

Активируется профилем `console`.

#### External-Ui

REST API для внешнего графического интерфейса, который может быть написан на любом языке (Python, C++, JavaScript и
т.д.).

**Эндпоинты:**

- `GET /api/patterns` — получение паттернов
- `POST /api/trade` — выполнение сделки
- `GET /api/portfolio` — информация о портфеле
- `WebSocket /api/quotes` — поток котировок в реальном времени

**Технологии:** Spring Web, WebSocket, Jackson.

Активируется профилем `external`. При активации поднимается встроенный веб-сервер (Tomcat), и API становится доступным
по HTTP.

### App (сборочный модуль)

Содержит главный класс `@SpringBootApplication` и конфигурационные файлы. Зависит от всех остальных модулей и собирает
итоговый исполняемый JAR.

## 🎛 Профили запуска

Приложение поддерживает два профиля, определяющих способ взаимодействия с пользователем:

| Профиль    | Описание                                                                                                                 |
|------------|--------------------------------------------------------------------------------------------------------------------------|
| `console`  | Запуск в консольном режиме. Используется `console-ui`.                                                                   |
| `external` | Запуск с поддержкой внешнего GUI. Поднимается REST API (`external-ui`) для взаимодействия с интерфейсом на другом языке. |

**Активация профиля:**

```bash
# Консольный режим
java -jar app/target/trading-app.jar --spring.profiles.active=console

# Режим с REST API (для внешнего GUI)
java -jar app/target/trading-app.jar --spring.profiles.active=external
После запуска в режиме external REST API будет доступно по адресу http://localhost:8080/api/... (порт настраивается).
```

## 💾 База данных

Используется **единая база данных PostgreSQL** для хранения всей информации:

| Таблица       | Назначение                             | Модуль                    |
|---------------|----------------------------------------|---------------------------|
| `trades`      | Реальные сделки                        | `core/trader`             |
| `test_trades` | Тестовые (симулированные) сделки       | `core/test-trader`        |
| `patterns`    | Обнаруженные торговые паттерны         | `analyzer`                |
| `quotes`      | Исторические и текущие котировки (кэш) | `market-data`, `analyzer` |
| `settings`    | Настройки приложения                    | все модули                |

## ⚙️ Установка и запуск

### Требования

- Java 17 или новее
- PostgreSQL 15
- Maven (для сборки)
PostgreSQL 15

Maven (для сборки)

Установка
Клонируйте репозиторий:

```bash
git clone https://github.com/IljiaPolstianoy/Trading.git
cd Trading
```

Создайте базу данных:

```sql
CREATE DATABASE trading;
```

Настройте параметры подключения в app/src/main/resources/application.yml (см. раздел Конфигурация)

Соберите проект:

```bash
mvn clean package
```

Запуск
```bash
# Консольный режим
java -jar app/target/trading-app.jar --spring.profiles.active=console

# Режим с REST API (для внешнего GUI)
java -jar app/target/trading-app.jar --spring.profiles.active=external
```

👨‍💻 Разработка
Сборка и тестирование

```bash
# Сборка всех модулей
mvn clean compile

# Запуск тестов
mvn test

# Сборка исполняемого JAR
mvn package
```

Добавление новой биржи/брокера
Создать подмодуль в exchange-connector (например, exchange-connector-new)

Создать подмодуль в market-data (например, market-data-new)

Добавить пакеты в trader и test-trader с реализациями, использующими новые клиенты

Добавить конфигурацию в application.yml
