**Trading** — многомодульное Java-приложение для автоматической и тестовой торговли ценными бумагами через API бирж и
брокеров с аналитикой и автоматическим выявлением новых торговых паттернов.

> **Текущий статус:** Реализована поддержка биржи ByBit (API v1.4.0) и торговой пары BTCUSDT. Консольный режим работы —
> готов. REST API для внешнего GUI, поддержка других бирж/брокеров и других ценных бумаг — в планах.

## 🎯 О проекте

**Trading** решает три ключевые задачи:

| № | Проблема                                                                                                                                                         | Решение                                                                                                            |
|---|------------------------------------------------------------------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------|
| 1 | **Риск потери средств** при тестировании стратегий                                                                                                               | Тестовый режим — симуляция сделок на реальных данных без отправки ордеров на биржу/брокеру                         |
| 2 | **Отсутствие автоматизации** — трейдеры тратят время на ручной анализ и выставление ордеров                                                                      | Полноценный торговый бот — автоматическое выявление паттернов и выставление ордеров                                |
| 3 | **Сложность поиска новых стратегий** — аналитикам приходится вручную перебирать комбинации индикаторов, а паттерны ведут себя по-разному для разных ценных бумаг | Автоматическое выявление **новых, ранее неизвестных паттернов** с учётом специфики каждой конкретной ценной бумаги |

**Ключевая особенность:** приложение не просто ищет известные паттерны, а **выявляет новые, ранее неизвестные
закономерности**, анализируя поведение конкретной ценной бумаги. Для одной бумаги один паттерн может встречаться чаще,
для другой — совершенно иной. Приложение учитывает эту индивидуальность.

> **Поддерживаемые ценные бумаги:** архитектура рассчитана на любые ценные бумаги (криптовалюты, акции, облигации,
> фьючерсы и т.д.). На данный момент реализована поддержка торговой пары **BTCUSDT**.

## ✨ Возможности

- 💰 **Реальная торговля** — через официальный API поддерживаемых бирж/брокеров (на данный момент — ByBit API v1.4.0,
  BTCUSDT)
- 🧪 **Тестовая торговля** — симуляция сделок без отправки ордеров на биржу/брокеру
- 🔍 **Анализ и выявление паттернов** — поиск и выявление торговых паттернов в реальном времени
- 🧠 **Выявление новых паттернов** — автоматическое обнаружение ранее неизвестных закономерностей для каждой конкретной
  ценной бумаги
- 📊 **Анализ индивидуальности бумаг** — система учитывает, что для разных активов работают разные паттерны
- 🖥️ **Консольный интерфейс** — управление приложением через командную строку (реализован)
- 🌐 **REST API для внешнего GUI** — возможность подключения внешнего интерфейса на любом языке (в планах)
- 🗄️ **Единая БД** — PostgreSQL для хранения всех событий
- 🔌 **Расширяемость** — архитектура позволяет добавлять поддержку новых бирж, брокеров и ценных бумаг
- 📝 **Log4j2** — гибкое логирование с возможностью настройки

### Установка и настройка

```bash
# 1. Клонируйте репозиторий
git clone https://github.com/IljiaPolstianoy/Trading.git
cd Trading

# 2. Создайте базу данных
psql -U postgres -c "CREATE DATABASE trading;"

# 3. Создайте ENUM типы и таблицы
psql -U postgres -d trading -f schema.sql
```

> Файл `schema.sql` содержит DDL скрипты для создания ENUM типов (`symbol`, `side`, `order_type`) и таблиц (
`test_orders`, `pattern`, `pattern_candle_directions` и др.).

### Настройка подключения к БД

Отредактируйте `app/src/main/resources/application.yml`:

```yaml
spring:
  datasource:
    driverClassName: org.postgresql.Driver
    url: jdbc:postgresql://localhost:5432/trading
    username: dbuser        # замените на вашего пользователя
    password: 12345         # замените на ваш пароль
```

### Сборка и запуск

```bash
# Сборка проекта
mvn clean package

# Запуск в консольном режиме
java -jar app/target/trading-app.jar --spring.profiles.active=console
```

> **В планах:** режим с REST API для внешнего GUI (`--spring.profiles.active=external`)

## 🧩 Модули

### Common

Общие компоненты для всех модулей: интерфейсы, DTO, утилиты, константы, настройки БД, исключения.

### Exchange-Connector

**Назначение:** фасад для работы с API бирж и брокеров. Модуль скрывает от пользователя всю сложность создания и
настройки клиентов для каждого конкретного брокера/биржи. Взаимодействие с любым подключённым брокером происходит через
единый интерфейс.

| Подмодуль | Назначение                                                                                                                                                                                         | Статус       |
|-----------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|--------------|
| `core`    | **Фасад** — единый интерфейс для всех брокеров/бирж. Абстрактные классы, обработка ошибок, ретраи. Пользователь работает только через этот модуль, не зная деталей реализации конкретного брокера. | ✅ Реализован |
| `bybit`   | **Реализация фасада** — создание и настройка клиента для ByBit (официальный Java SDK v1.4.0). Вся специфика ByBit скрыта внутри этого подмодуля.                                                   | ✅ Реализован |
| `tbank`   | **Реализация фасада** — создание и настройка клиента для Т-банка (брокерский API). В планах.                                                                                                       | 📋 В планах  |

### Market-Data

**Назначение:** фасад для получения рыночных данных (котировки, свечи, стаканы). Модуль скрывает от пользователя
особенности получения данных от разных источников (бирж и брокеров). Архитектура поддерживает любые ценные бумаги.

| Подмодуль | Назначение                                                                                                                                                                         | Статус       |
|-----------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|--------------|
| `core`    | **Фасад** — единый интерфейс для получения рыночных данных от любых источников. Абстрактные классы, кэширование, обработка ошибок. Пользователь работает только через этот модуль. | ✅ Реализован |
| `bybit`   | **Реализация фасада** — получение котировок от ByBit (поддерживает BTCUSDT). Детали подключения к WebSocket/REST API ByBit скрыты.                                                 | ✅ Реализован |
| `tbank`   | **Реализация фасада** — получение котировок от Т-банка. В планах.                                                                                                                  | 📋 В планах  |

### Core (ядро приложения)

**Trader** (реальная торговля) — получает котировки от `market-data`, анализирует выявленные паттерны, формирует и
отправляет ордера через `exchange-connector`.

**Test-Trader** (тестовая торговля) — получает те же данные, но не отправляет реальные ордера. Симулирует сделки и
записывает их в таблицу `test_trades`. Позволяет оценить эффективность выявленных паттернов без финансового риска.

### Analyzer

Анализирует поток котировок в реальном времени, **выявляет как известные, так и совершенно новые, ранее неизвестные
паттерны**.

**Ключевые возможности анализатора:**

- **Индивидуальный подход к каждой ценной бумаге** — система анализирует специфику каждого актива и выявляет паттерны,
  характерные именно для него
- **Автоматическое выявление новых паттернов** — специальный метод анализирует исторические данные, перебирает
  комбинации индикаторов и временных интервалов, выявляет статистически значимые корреляции
- **Сравнительный анализ** — система может сравнивать поведение разных бумаг и выявлять, какие паттерны работают для
  каких активов

### Interaction (модули взаимодействия)

**UI-API (фасад)** — предоставляет единый набор сервисов для всех типов интерфейсов. Все UI-модули зависят только от
`ui-api`, что изолирует их от деталей реализации ядра.

**Console-UI** — консольный интерфейс. ✅ **Реализован**

| Возможность                          | Команда |
|--------------------------------------|---------|
| Поиск и выявление паттернов          | `1`     |
| Запуск тестовой торговли (симуляция) | `2`     |
| Запуск реальной торговли (на бирже)  | `3`     |
| Завершение работы                    | `4`     |

**External-UI** — REST API для внешнего графического интерфейса. 📋 **В планах**

### App (сборочный модуль)

Содержит `TradingApplication.java` и конфигурационные файлы. Зависит от всех модулей, собирает итоговый JAR.

## 💾 База данных

**PostgreSQL** (последняя версия, на данный момент 18.3) — единая БД для всей информации.

### Схема базы данных

```sql
-- ENUM типы
CREATE TYPE symbol AS ENUM ('BTCUSDT');
CREATE TYPE side AS ENUM ('BUY', 'SELL');
CREATE TYPE order_type AS ENUM ('MARKET', 'LIMIT');

-- Таблицы
CREATE TABLE IF NOT EXISTS test_orders
(
    id         BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    symbol     VARCHAR         NOT NULL,
    side       VARCHAR         NOT NULL,
    order_type VARCHAR         NOT NULL,
    price      DECIMAL(10, 2)  NOT NULL,
    amount     DECIMAL(30, 20) NOT NULL
);

CREATE TABLE IF NOT EXISTS pattern
(
    id SERIAL PRIMARY KEY
);

CREATE TABLE IF NOT EXISTS pattern_candle_directions
(
    pattern_id        INTEGER NOT NULL,
    position          INTEGER NOT NULL,
    candle_directions BOOLEAN NOT NULL,
    FOREIGN KEY (pattern_id) REFERENCES pattern (id),
    PRIMARY KEY (pattern_id, position)
);
```

### Таблицы (расширенный список)

| Таблица                     | Назначение                    | Модуль             |
|-----------------------------|-------------------------------|--------------------|
| `test_orders`               | Тестовые ордера (симуляция)   | `core/test-trader` |
| `pattern`                   | Выявленные паттерны           | `analyzer`         |
| `pattern_candle_directions` | Направления свечей паттерна   | `analyzer`         |
| `trades`                    | Реальные сделки (планируется) | `core/trader`      |
| `quotes`                    | Котировки (планируется)       | `market-data`      |
| `settings`                  | Настройки (планируется)       | все модули         |
