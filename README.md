# Trading Application

Многомодульное Java-приложение для автоматической и тестовой торговли криптовалютами на бирже ByBit с использованием
аналитики паттернов.

## 📋 О проекте

**Trading Application** — это настольное приложение для автоматической торговли Bitcoin на бирже ByBit.
Система анализирует рыночные данные, выявляет торговые паттерны и на их основе совершает сделки.
Ключевая особенность — наличие **тестового режима**,
позволяющего проверять эффективность паттернов без риска потери реальных средств.

### Возможности

- Реальная торговля на ByBit через официальный API
- Тестовая торговля на исторических и реальных данных (без отправки ордеров на биржу)
- Анализ рыночных данных и поиск торговых паттернов
- Два режима работы: консольный и с графическим интерфейсом
- Единая база данных для хранения всех событий

## 🏗 Архитектура

Приложение построено как **модульный монолит**: один процесс,
одна JVM с чётким разделением на функциональные модули внутри одного приложения.
Это позволяет в будущем выделить модули в микросервисы без переписывания логики.

## 🧩 Модули

### Core (ядро приложения)

#### Trading

Модуль реальной торговли. Содержит подмодули для каждой поддерживаемой биржи. Взаимодействует с биржевыми API,
отправляет реальные ордера. Записывает информацию о всех совершённых сделках в таблицу `trades`.

**Структура:**

- `trading/common` — общие интерфейсы и утилиты для всех бирж
- `trading/bybit` — реализация для ByBit (официальный Java SDK, REST/WebSocket клиенты)

#### TestTrading

Модуль тестовой торговли. Получает те же рыночные данные, что и реальный модуль, и принимает те же торговые решения, но
**не отправляет запросы на биржи**. Вместо этого симулирует сделки и записывает их в отдельную таблицу `test_trades`.
Позволяет оценить эффективность паттернов без финансового риска.

**Структура:**

- `testtrading/common` — общие интерфейсы для тестовой торговли
- `testtrading/bybit` — симуляция торговли на ByBit

**Технологии:** мокирование API, отдельные сервисы-заглушки.

#### Common

Общие компоненты для всех модулей ядра: DTO, утилиты, константы, настройки подключения к БД.

### Analyzer

Аналитический модуль. В реальном времени анализирует поток котировок, выявляет заданные торговые паттерны и сохраняет
информацию о них в базу данных. Модуль не зависит от торговых модулей и работает при запуске из интерфейса.

**Технологии:** многопоточность (`@Async`), паттерны анализа данных.

### Interaction (модули взаимодействия с пользователем)

#### Console (профильный модуль)

Консольный интерфейс для управления приложением. Запускается при активном профиле `console`. Позволяет просматривать
текущие котировки, включать/отключать торговые модули, просматривать статистику по сделкам и паттернам, управлять
настройками анализатора.

**Технологии:** Spring Shell или простой Scanner.

#### UI (профильный модуль)

Графический интерфейс (на другом языке, общается с Java-ядром по HTTP). Запускается при активном профиле `ui`.
Предоставляет те же возможности, что и консольный модуль, но в удобном визуальном формате.

**Технологии:** встроенный веб-сервер (Spring MVC), REST API, HTML/JS (или другой язык).

## 🎛 Профили запуска

Приложение поддерживает два профиля, определяющих способ взаимодействия с пользователем:

| Профиль   | Описание                                                                                                |
|-----------|---------------------------------------------------------------------------------------------------------|
| `console` | Запуск в консольном режиме. Весь интерфейс — командная строка.                                          |
| `ui`      | Запуск с графическим интерфейсом. Поднимается встроенный веб-сервер и REST API для взаимодействия с UI. |

**Активация профиля:**

```bash
# Консольный режим
java -jar trading-bot.jar --spring.profiles.active=console

# Режим с GUI
java -jar trading-bot.jar --spring.profiles.active=gui
```

## 💾 База данных

Используется **единая база данных PostgreSQL** для хранения всей информации:

| Таблица       | Назначение                             | Модуль           |
|---------------|----------------------------------------|------------------|
| `trades`      | Реальные сделки, отправленные на биржу | trading          |
| `test_trades` | Тестовые (симулированные) сделки       | testtrading      |
| `patterns`    | Обнаруженные торговые паттерны         | analyzer         |
| `quotes`      | Исторические и текущие котировки (кэш) | analyzer/trading |
| `settings`    | Настройки приложения                   | все модули       |

## ⚙️ Установка и запуск

### Требования

Java 25 или новее

PostgreSQL 15

Maven (для сборки)

### Установка

Клонируйте репозиторий:

```bash
git clone https://github.com/IljiaPolstianoy/Traiding.git
cd Trading
```

Создайте базу данных:

```sql
CREATE
DATABASE trading;
```

Настройте параметры подключения в application.properties (см. раздел Конфигурация)

### Соберите проект:

```bash
mvn clean package
```

### Запуск

```bash
# Консольный режим
java -jar target/trading.jar --spring.profiles.active=console

# Режим с GUI
java -jar target/trading-bot.jar --spring.profiles.active=ui
```

## 🔧 Конфигурация

Основные настройки в application.properties:

```properties
# База данных
spring.datasource.url=jdbc:postgresql://localhost:5432/trading
spring.datasource.username=dbuser
spring.datasource.password=12345
# ByBit API (для продакшен-режима)
bybit.api.key=your_api_key
bybit.api.secret=your_api_secret
bybit.api.testnet=false # true для тестовой сети
# Многопоточность
spring.task.execution.pool.core-size=5
spring.task.execution.pool.max-size=20
```
